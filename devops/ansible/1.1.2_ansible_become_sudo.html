1.1.2: ansible-become(sudo)

1. why should use become

We don't use root directly in ansible in most cases. Instead, we prefer a common user to execute some tasks in sudo mode. By doing so, we can keep the security.

2. how we use become

Let's make an example, we try to manage tomcat init script by using ansible become

1) first create a common user and switch to it

# on ansible control node
useradd classmate
su - classmate

# on controled node
useradd classmate

实际上控制端不需要有classmate这个用户而使用root也可以，但是为了模仿线上环境，我们同样创建一个classmate用户

2) yaml file content

---
- hosts: test-host
  remote_user: classmate
  become: true
  become_user: root
  become_method: sudo

  tasks:
   - name: start tomcat1
     service: name=tomcat1 state=started
---
